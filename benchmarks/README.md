# php-bloom benchmarks

This directory contains two benchmark suites:

- `benchmarks/bench.php` compares the `Bloom\Filter` C extension with a pure
  PHP baseline.
- `benchmarks/mysql/bench.php` compares direct indexed MySQL lookups with a
  `Bloom\Filter` pre-check for suppression-style queries.

## Extension vs pure PHP

Build the extension first:

```sh
scripts/build.sh
```

Then run the default benchmark:

```sh
php -d extension=modules/bloom.so benchmarks/bench.php
```

The default configuration is:

- capacity: `100000`
- false positive rate: `0.01`
- inserts: `100000`
- present lookups: `100000`
- absent lookups: `100000`

You can override sizes with CLI flags:

```sh
php -d extension=modules/bloom.so benchmarks/bench.php --capacity=1000000 --inserts=1000000 --lookups=1000000
```

Or with environment variables:

```sh
BLOOM_BENCH_CAPACITY=1000000 \
BLOOM_BENCH_INSERTS=1000000 \
BLOOM_BENCH_LOOKUPS=1000000 \
php -d extension=modules/bloom.so benchmarks/bench.php
```

JSON output is available for future comparison or CI usage:

```sh
php -d extension=modules/bloom.so benchmarks/bench.php --json
```

If the extension is not loaded, the default benchmark refuses to run because
it cannot compare both implementations. The pure PHP baseline can still be run
independently:

```sh
php benchmarks/bench.php --impl=pure
```

## What is measured

The benchmark measures:

- constructing the filter
- adding deterministic generated values
- checking known-present values
- checking known-absent values
- `export()`
- `import()`
- `stats()`, when present

Each operation records elapsed time from `hrtime(true)`, operations per second,
nanoseconds per operation, memory usage delta, and operation-specific result
metadata such as exported binary size or false positives.

## Pure PHP baseline

`PurePhpBloomFilter.php` intentionally mirrors the extension as closely as
practical:

- same capacity and false-positive-rate sizing formulas
- same hash round count formula
- same seeded 64-bit FNV-1a hash pair
- same unsigned 64-bit wraparound during double hashing
- same little-endian `BLM1` export format
- binary-safe PHP strings for input values
- a binary string bitset, not an array of booleans

Input values are deterministic strings generated by the benchmark. No external
files, random data, or real user data are used.

## MySQL suppression lookup benchmark

The MySQL benchmark models a common production shape: an authoritative
suppression table backed by an indexed lookup, with a Bloom filter used to avoid
database queries for values that are definitely absent.

Start MySQL with the benchmark Compose file:

```sh
docker compose -f benchmarks/docker-compose.yml up -d
```

Seed the default deterministic dataset:

```sh
php benchmarks/mysql/seed.php
```

Run the benchmark:

```sh
php -d extension=modules/bloom.so benchmarks/mysql/bench.php
```

The default MySQL benchmark compares:

- direct MySQL lookups
- Bloom filter pre-check plus MySQL verification
- Bloom filter only, as a raw lookup ceiling

The default dataset uses `100000` deterministic SHA-256 suppression hashes,
`100000` checks per scenario, a `0.01` Bloom false-positive rate, and multiple
present/absent mixes.

If local port `3306` is already in use, choose another port and pass it to both
the seed and benchmark commands:

```sh
MYSQL_PORT=3307 docker compose -f benchmarks/docker-compose.yml up -d
php benchmarks/mysql/seed.php --port=3307
php -d extension=modules/bloom.so benchmarks/mysql/bench.php --port=3307
```

Larger datasets can be seeded and benchmarked explicitly:

```sh
php benchmarks/mysql/seed.php --dataset=1000000
php -d extension=modules/bloom.so benchmarks/mysql/bench.php --dataset=1000000 --checks=1000000
```

JSON output is also available:

```sh
php -d extension=modules/bloom.so benchmarks/mysql/bench.php --json
```

See `benchmarks/mysql/README.md` for detailed MySQL benchmark configuration,
data model, and result interpretation.
